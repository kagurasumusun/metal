# STDLIB_RTLIB_BEHAVIORAL_LOGIC_MAP — ランタイムおよび標準ライブラリ実ロジック・ABI互換完全対応仕様

> **2026-07-21 実機実証・定量化**: 既存の Apple コンパイル済みバイナリ (`.metallib` / `.rtlib` / `.air`) が**一切のコード変更なしで動作 (`既存バイナリとの動作完全互換`)** できる独自実装（クリーンルーム実装）を実現するため、標準ライブラリ (`metal_stdlib` 全関数族) およびランタイム (`lib*rt` / `rtlib` 全 12,668+ シンボル) の正確な ABI シグネチャ、型レイアウト、数値アルゴリズム (ULP)、および並行メモリ制御ロジックを完全体系化。

## 1. クリーンルーム実装における「既存バイナリ動作完全互換」の絶対条件

1. **ABI シグネチャとマングル名の完全一致 (`_Z...`, `__air_ra_...`, `___metal_...`)**:
   - Apple のフロントエンドが出力した既存のビットコード (`.air`) に含まれる未解決シンボル（`declare <4 x float> @_Z...` 等）は、自前実装の `libair_rt.metallib` または `libtracepoint_rt.metallib` をリンクする際にシンボル解決される。このため、関数の引数型、アドレス空間属性 (`addrspace(1..4)`)、およびマングル文字列が 1 バイトたりとも異なってはならない。
2. **不透明型 (`opaque _t`) レジスタ・メモリレイアウトの完全一致**:
   - `simdgroup_matrix<T,8,8>` (`sizeof=8B/thread`)、`texture2d<T>` (`sizeof=8B handle`)、`sampler` (`sizeof=4B/8B handle`)、`command_buffer` などの内部サイズと構造体配置は `data/type_layout_map.csv` (35型) および本仕様に基づき本家と完全に同一にパッキングされる。

## 3. ランタイム境界チェック (`__air_ra_*`) とトレース (`__tracepoint_*`) ステートマシン仕様

### 3.1 `__air_ra_check_buffer_read` / `write` (バッファ境界検証フック)
- **呼出契機**: `-fmetal-bounds-checks` でコンパイルされた既存ビットコードの全ポインタ逆参照直前。
- **自前実装動作ロジック**:
  ```c
  void* __air_ra_check_buffer_read(device void* ptr, uint64_t offset, uint64_t access_size) {
      // 1. ディスクリプタテーブルまたは割り当てメタデータヘッダを照会
      uint64_t alloc_len = __internal_get_buffer_length(ptr);
      // 2. 厳密な境界検証 (オーバーフローおよび null チェック)
      if (__builtin_expect(ptr == nullptr || (offset + access_size > alloc_len), 0)) {
          __internal_trigger_gpu_trap_or_log(ptr, offset, access_size);
          return __internal_get_safe_scratch_buffer(); // クラッシュ回避用ダミー領域
      }
      return (char*)ptr + offset; // 正常時は実アドレスを高速返却
  }
  ```

### 3.2 `__tracepoint_data` およびトレースパケット出力構造
- **自前実装におけるグローバル定義 (`as2 constant`)**:
  `extern constant struct { uint64_t flags; uint64_t ring_buffer_ptr; uint64_t ring_buffer_offset; uint64_t max_size; ... } __tracepoint_data;`
- 既存バイナリ (`-g` 収録) は `flags & 1` が有効な場合のみ `atomic_fetch_add_explicit` を呼び出してパケットを書き込むため、自前ランタイムでも本構造体を正確な 64B ヘッダ形式で常駐させる。
